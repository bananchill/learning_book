# Операторы сравнения

Сравнение — одна из самых частых операций в коде: условия, фильтрация, сортировка. JavaScript предоставляет два вида сравнения на равенство — **строгое** (`===`) и **нестрогое** (`==`). Они ведут себя принципиально по-разному, и понимание разницы поможет избежать множества неочевидных багов.

## Строгое сравнение ===

`===` — самый предсказуемый оператор сравнения. Он проверяет и тип, и значение — без каких-либо преобразований. Если типы операндов различаются, результат сразу `false`:

```javascript
1 === 1       // true
1 === '1'     // false — разные типы
null === null // true
NaN === NaN   // false — NaN не равен ничему, включая себя!
```

Особняком стоит `NaN` — единственное значение в JavaScript, которое не равно самому себе. Это стандарт IEEE 754 для чисел с плавающей точкой: `NaN` означает «результат бессмысленной математической операции» (например, `0 / 0`), и сравнивать такие результаты между собой нет смысла. Для проверки используйте `Number.isNaN()`:

```javascript
Number.isNaN(NaN)          // true
Number.isNaN(undefined)    // false — в отличие от глобального isNaN()
Number.isNaN('hello')      // false
```

## Нестрогое сравнение ==

`==` перед сравнением пытается привести операнды к общему типу. Набор правил приведения сложен и неинтуитивен — именно поэтому результат часто удивляет даже опытных разработчиков:

```javascript
1 == '1'      // true
0 == false    // true
0 == ''       // true
null == undefined // true — единственная пара
[] == false   // true ([] → '' → 0, false → 0)
'' == false   // true
```

<Callout type="warning">
Используйте `===` и `!==` по умолчанию. Исключение: `value == null` — удобная проверка на null ИЛИ undefined.
</Callout>

## Операторы отношения

Операторы `>`, `<`, `>=`, `<=` сравнивают два значения и возвращают `true` или `false`. С числами всё предсказуемо, но со строками работает **лексикографическое сравнение** — посимвольно, по кодам Unicode. Это значит, что заглавные буквы «меньше» строчных (потому что `'A'` = 65, `'a'` = 97), а строка `'10'` «меньше» `'9'`, потому что первый символ `'1'` имеет код меньше `'9'`:

```javascript
5 > 3    // true
5 < 3    // false
5 >= 5   // true
5 <= 4   // false

// Строки сравниваются лексикографически (по Unicode)
'abc' < 'abd'  // true — 'c' < 'd'
'B' < 'a'      // true — 'B' (66) < 'a' (97)
'10' < '9'     // true — строковое сравнение! '1' < '9'

// Смешанные типы — строка приводится к числу
'10' < 9      // false — 10 > 9
```

Когда сравниваются значения разных типов (строка и число), JavaScript приводит строку к числу. Именно поэтому `'10' < 9` даёт `false` — строка `'10'` становится числом `10`.

## Сравнение с null и undefined

Это одна из самых запутанных тем. Операторы `>`, `<`, `>=`, `<=` приводят `null` к `0`, а `undefined` к `NaN`. Но оператор `==` использует особые правила: `null == undefined` — `true`, а `null == 0` — `false`. Эти два набора правил противоречат друг другу, поэтому результат выглядит нелогично:

```javascript
null > 0   // false
null == 0  // false
null >= 0  // true  ← неожиданно!

// Причина: >= приводит null к 0, но == использует особое правило
// null == только null или undefined

undefined > 0   // false
undefined < 0   // false
undefined == 0  // false — undefined только == null
```

<Callout type="info">
Избегайте сравнений `>`, `<`, `>=`, `<=` с `null` и `undefined`. Поведение нелогично и часто приводит к багам.
</Callout>

<DeepDive title="Как работает == под капотом">

Алгоритм Abstract Equality Comparison из ECMAScript:
1. Если типы одинаковые — используется ===
2. `null == undefined` → true
3. `undefined == null` → true
4. Если один number, другой string → string приводится к number
5. Если один boolean → boolean приводится к number
6. Если один object, другой primitive → вызывается ToPrimitive(object)

Именно поэтому `[] == false`:
- `[]` → `''` (ToPrimitive)
- `false` → `0`
- `''` → `0`
- `0 === 0` → true

</DeepDive>
