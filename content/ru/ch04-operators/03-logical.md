# Логические операторы

Логические операторы в JavaScript работают не так, как в большинстве других языков. Они не просто возвращают `true` или `false` — они возвращают **одно из значений-операндов**. Это позволяет использовать их не только в условиях, но и для установки значений по умолчанию, условного выполнения и безопасного доступа к свойствам.

Ключевая идея — **short-circuit evaluation** (вычисление с коротким замыканием). Оператор прекращает вычисление, как только результат становится очевидным: `&&` останавливается на первом `false`, `||` — на первом `true`. Правая часть выражения просто не выполнится, если результат уже определён левой.

## && — логическое И

`&&` возвращает первый falsy-операнд, или последний, если все truthy. Благодаря short-circuit правая часть не вычисляется, если левая уже falsy:

```javascript
// Базовое использование
true && true    // true
true && false   // false
false && true   // false

// Short-circuit: если левая часть falsy — правая не вычисляется
false && expensiveOperation()  // expensiveOperation не вызовется!

// Возвращает значение, не просто true/false
'hello' && 'world'   // 'world' — обе truthy, вернул последнюю
null && 'world'      // null — первая falsy, вернул её
0 && 'world'         // 0
```

## || — логическое ИЛИ

`||` — зеркальная противоположность `&&`. Он возвращает первый truthy-операнд, или последний, если все falsy. Это делает его идеальным для установки значений по умолчанию — но с важной оговоркой:

```javascript
true || false    // true
false || true    // true
false || false   // false

// Классическое применение: значение по умолчанию
const name = userInput || 'Аноним';  // 'Аноним' если userInput falsy
const port = config.port || 3000;

// Ловушка: 0 и '' — тоже falsy!
const count = userCount || 10;  // Если userCount = 0, получим 10 — неправильно!
```

<Callout type="warning">
Используйте `||` для значений по умолчанию только если 0 и пустая строка недопустимы. Иначе используйте `??`.
</Callout>

## ?? — оператор нулевого слияния (ES2020)

Оператор `??` появился именно для решения проблемы `||` с falsy-значениями. Если `||` срабатывает на любом falsy (`0`, `''`, `false`, `null`, `undefined`), то `??` реагирует **только** на `null` и `undefined` — то есть на «значение отсутствует», а не на «значение пустое»:

```javascript
null ?? 'default'       // 'default'
undefined ?? 'default'  // 'default'
0 ?? 'default'          // 0 — не заменяется! 0 не null/undefined
'' ?? 'default'         // '' — не заменяется!
false ?? 'default'      // false — не заменяется!

// Правильный выбор для числовых настроек
const timeout = options.timeout ?? 3000;  // 0 — валидное значение!
const label = options.label ?? 'Default'; // '' — может быть валидно
```

## ! — логическое НЕ

Оператор `!` инвертирует truthiness значения. Двойное отрицание `!!` — распространённый приём для явного приведения любого значения к `boolean`:

```javascript
!true   // false
!false  // true
!0      // true — falsy-значение
!''     // true
!null   // true

// Двойное отрицание — приведение к boolean
!!0     // false
!!''    // false
!!42    // true
!!'hi'  // true
!!null  // false
```

## Практические паттерны

Благодаря short-circuit логические операторы часто используются как компактная замена `if`:

```javascript
// Условное выполнение
isLoggedIn && showDashboard();  // вместо if (isLoggedIn) showDashboard()

// Опциональное свойство (ES2020: ?.)
const city = user?.address?.city ?? 'Неизвестно';
```

**Logical assignment** (ES2021) — это сокращённая форма записи «установи значение, если текущее не подходит». Оператор `||=` записывает значение, если текущее falsy. `??=` — только если текущее `null` или `undefined`. `&&=` — только если текущее truthy (полезно для обновления существующих значений):

```javascript
config.host ||= 'localhost';   // если host falsy — установить localhost
config.timeout ??= 3000;       // если timeout null/undefined — установить 3000
config.name &&= config.name.trim(); // если name truthy — обрезать пробелы
```

<DeepDive title="Порядок выполнения логических операторов">

Приоритет (от высшего к низшему):
1. `!` (НЕ)
2. `&&` (И)
3. `||` (ИЛИ)
4. `??` (нулевое слияние)

```javascript
// Внимание: нельзя смешивать ?? с || и && без скобок!
a || b ?? c   // SyntaxError в большинстве случаев
(a || b) ?? c // OK — явные скобки
```

</DeepDive>
