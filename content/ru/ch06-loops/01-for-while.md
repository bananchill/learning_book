# for, while, do...while

Циклы позволяют выполнять один и тот же блок кода многократно. Без них пришлось бы вручную копировать строки для каждого элемента массива или каждого шага вычисления. JavaScript предлагает три базовых цикла, и каждый подходит для своей ситуации.

## Цикл for

Классический цикл со счётчиком — используйте, когда заранее известно, сколько раз нужно повторить действие. Он состоит из трёх частей: **инициализация** (выполняется один раз перед началом), **условие** (проверяется перед каждой итерацией) и **обновление** (выполняется после каждой итерации):

```javascript
// Синтаксис: for (инициализация; условие; обновление)
for (let i = 0; i < 5; i++) {
  console.log(i); // 0, 1, 2, 3, 4
}

// Обратный отсчёт
for (let i = 10; i >= 0; i--) {
  console.log(i);
}

// Итерация по массиву с индексом
const fruits = ['яблоко', 'банан', 'вишня'];
for (let i = 0; i < fruits.length; i++) {
  console.log(`${i}: ${fruits[i]}`);
}
```

Все три части `for` необязательны:

```javascript
let i = 0;
for (; i < 3;) {   // пустая инициализация и обновление
  console.log(i);
  i++;
}

// Бесконечный цикл (выход через break)
for (;;) {
  if (someCondition) break;
  doWork();
}
```

## Цикл while

`while` — цикл без счётчика. Используйте когда количество итераций заранее неизвестно и зависит от условия: чтение данных до конца, ожидание ответа, поиск нужного значения. Цикл проверяет условие **перед** каждой итерацией — если оно изначально ложно, тело не выполнится ни разу:

```javascript
// Чтение до конца данных
let line = readLine();
while (line !== null) {
  process(line);
  line = readLine();
}

// Числовой пример
let n = 1;
while (n < 100) {
  n *= 2;
}
console.log(n); // 128 — первое число >= 100, которое степень 2
```

<Callout type="warning">
Убедитесь, что в теле цикла `while` условие когда-нибудь становится ложным — иначе бесконечный цикл заморозит вкладку браузера.
</Callout>

## Цикл do...while

Единственное отличие от `while` — проверка условия происходит **после** итерации, а не до. Поэтому тело гарантированно выполнится хотя бы один раз. Это удобно для запросов ввода (сначала спроси — потом проверь) и обработки данных, которые нужно прочитать хотя бы раз:

```javascript
// Запрос ввода — выполняется хотя бы раз
let userInput;
do {
  userInput = prompt('Введите число от 1 до 10');
} while (userInput < 1 || userInput > 10);

// Обработка с гарантированным первым выполнением
let result = 0;
let value = getValue();
do {
  result += value;
  value = getValue();
} while (value > 0);
```

## Когда что использовать

| Цикл | Когда использовать |
|---|---|
| `for` | Известное количество итераций, нужен индекс |
| `while` | Неизвестное количество итераций, условие проверяется до |
| `do...while` | Нужно выполнить хотя бы раз |

<DeepDive title="for vs while — перформанс">

Современные JavaScript движки (V8) оптимизируют оба типа цикла практически одинаково. Разница незначительна. Выбирайте исходя из читаемости и семантики, не производительности.

Исключение: кэширование `arr.length` в `for` цикле исторически давало прирост, но сегодня V8 оптимизирует это автоматически. Тем не менее, явное кэширование делает намерение понятным.

</DeepDive>
