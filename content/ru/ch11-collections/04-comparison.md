# Когда что использовать

Четыре коллекции решают разные задачи. Выбор зависит от того, **что** хранится и **как долго** живут данные.

## Object vs Map vs WeakMap

Объект — привычный выбор для хранения пар «ключ → значение», но Map и WeakMap превосходят его в конкретных сценариях.

| Критерий | Object | Map | WeakMap |
|---|---|---|---|
| Тип ключей | строки, символы | любой тип | только объекты |
| Порядок ключей | числовые → строковые → символы | порядок вставки | — |
| Размер | `Object.keys(obj).length` | `map.size` | неизвестен |
| Итерация | `for...in`, `Object.keys()` | `for...of`, `forEach` | невозможна |
| Прототип | есть (нужна осторожность) | нет | нет |
| Производительность | быстрый для маленьких | быстрый для частых add/delete | как Map |
| Сборка мусора ключей | нет | нет | да |
| Сериализация (JSON) | нативная | ручная | невозможна |

Простой пример, показывающий разницу на практике.

```javascript
// Object — для известных заранее полей (конфиги, DTO)
const config = {
  theme: 'dark',
  lang: 'ru',
  fontSize: 16
};

// Map — для динамических ключей, особенно не строковых
const clickCount = new Map();
document.querySelectorAll('button').forEach(btn => {
  clickCount.set(btn, 0); // DOM-элемент как ключ
});

// WeakMap — для метаданных, которые не должны мешать сборке мусора
const metadata = new WeakMap();
function track(element) {
  metadata.set(element, { created: Date.now() });
  // Когда element удалится из DOM и потеряет ссылки — метаданные исчезнут
}
```

## Array vs Set vs WeakSet

Массив — универсальный список, но Set и WeakSet лучше подходят для работы с уникальными значениями.

| Критерий | Array | Set | WeakSet |
|---|---|---|---|
| Дубликаты | допускаются | автоматически исключены | автоматически исключены |
| Тип значений | любой | любой | только объекты |
| Доступ по индексу | да | нет | нет |
| Проверка наличия | `includes()` — O(n) | `has()` — O(1) | `has()` — O(1) |
| Итерация | полная | полная | невозможна |
| Размер | `.length` | `.size` | неизвестен |
| Сборка мусора | нет | нет | да |

Проверка наличия элемента — ключевое различие. У массива `includes()` перебирает элементы один за другим, а у Set `has()` работает мгновенно.

```javascript
// Array — для упорядоченных данных с дубликатами и индексами
const history = ['главная', 'каталог', 'главная', 'корзина'];

// Set — для уникальных значений и быстрой проверки принадлежности
const visited = new Set();
visited.add('главная');
visited.add('каталог');
visited.add('главная'); // дубликат — не добавится
console.log(visited.has('каталог')); // true — мгновенная проверка

// WeakSet — для отслеживания обработанных объектов
const sent = new WeakSet();
function sendNotification(user) {
  if (sent.has(user)) return; // уже отправляли
  sent.add(user);
  console.log(`Уведомление для ${user.name}`);
}
```

## Правила выбора

Несколько простых вопросов помогают выбрать правильную структуру.

```
Нужно хранить пары ключ-значение?
├── Ключи — строки, данные статичные → Object
├── Ключи — любые типы / частые add/delete → Map
└── Ключи — объекты, не нужна итерация, важна память → WeakMap

Нужно хранить набор значений?
├── Порядок важен, дубликаты допустимы → Array
├── Нужна уникальность / быстрый has() → Set
└── Значения — объекты, не нужна итерация → WeakSet
```

<Callout type="info">
Практическое правило: начинайте с Object/Array. Переключайтесь на Map/Set, когда появляется конкретная потребность — нестроковые ключи, уникальность, частые операции добавления и удаления. WeakMap/WeakSet — только когда важно не мешать сборщику мусора.
</Callout>
