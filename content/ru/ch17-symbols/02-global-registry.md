# Глобальный реестр символов

Обычные символы, созданные через `Symbol()`, уникальны и существуют только в текущей области видимости. Но иногда нужен один и тот же символ в разных частях программы — для этого существует глобальный реестр символов.

## Symbol.for(key)

Метод `Symbol.for(key)` ищет символ по ключу в глобальном реестре. Если символ с таким ключом уже существует — возвращает его. Если нет — создаёт новый, сохраняет в реестр и возвращает.

```javascript
// Первый вызов — создаёт символ и помещает в реестр
const s1 = Symbol.for('app.id');

// Второй вызов — находит существующий символ
const s2 = Symbol.for('app.id');

console.log(s1 === s2); // true — один и тот же символ
```

Ключ реестра — это строка, которая одновременно становится описанием символа. В отличие от `Symbol('описание')`, где описание — просто метка, ключ в `Symbol.for()` используется для поиска.

```javascript
const registered = Symbol.for('id');
console.log(registered.description); // 'id' — совпадает с ключом
```

## Symbol.keyFor(sym)

Обратная операция: принимает символ и возвращает его ключ в глобальном реестре. Если символ не зарегистрирован — возвращает `undefined`.

```javascript
const global = Symbol.for('token');
const local = Symbol('token');

console.log(Symbol.keyFor(global)); // 'token' — нашёл ключ
console.log(Symbol.keyFor(local));  // undefined — не в реестре
```

Это позволяет определить, был ли символ создан через `Symbol.for()` или через обычный `Symbol()`.

## Когда использовать глобальный реестр

Глобальный реестр полезен в нескольких сценариях. Главный из них — взаимодействие между разными частями приложения, которые не могут напрямую передавать друг другу ссылки на символы.

```javascript
// Файл: constants.js
// Библиотека регистрирует символ для межмодульного использования
const EVENT_TYPE = Symbol.for('mylib.eventType');

// Файл: handler.js
// Другой модуль получает тот же символ по ключу
const EVENT_TYPE = Symbol.for('mylib.eventType');

// Оба модуля работают с одним символом
```

Типичные случаи применения: взаимодействие между iframe и основным окном (кросс-реалмовое), интеграция между независимыми библиотеками, а также создание общих констант для плагинов.

<Callout type="info">
`Symbol()` и `Symbol.for()` — это разные механизмы. Обычный `Symbol('key')` НЕ помещает символ в глобальный реестр. `Symbol.for('key')` создаёт разделяемый символ, доступный из любой точки программы. Не путайте их — это частая ошибка.
</Callout>
