## Лексическое окружение (Lexical Environment)

Чтобы понять, как замыкания работают на самом деле, нужно знать одну вещь: каждый раз, когда вызывается функция, JavaScript-движок создаёт специальный внутренний объект — **Lexical Environment** (лексическое окружение). Этот объект хранит две вещи:

1. **Environment Record** — запись всех локальных переменных и параметров этого вызова
2. **[[OuterEnv]]** — ссылку на лексическое окружение, в котором функция была **создана** (не вызвана, а именно создана)

Когда JavaScript ищет переменную, он сначала смотрит в текущем Environment Record. Не нашёл — идёт по ссылке `[[OuterEnv]]` во внешнее окружение. Не нашёл там — идёт ещё выше. И так до глобального окружения. Если и там нет — `ReferenceError`.

```js
function outer() {
  let x = 10 // Environment Record: { x: 10 }

  function inner() {
    // [[OuterEnv]] → окружение outer
    console.log(x) // ищет x: нет локально → ищет в [[OuterEnv]] → находит 10
  }

  return inner
}
```

Именно благодаря ссылке `[[OuterEnv]]` внутренняя функция «помнит» переменные внешней — она буквально хранит указатель на окружение, в котором была создана.

## Scope chain по шагам

Посмотрим пошагово, что происходит при выполнении конкретного кода. Это поможет увидеть, как движок строит цепочку окружений и как замыкание находит нужные переменные.

```js
const greeting = 'Привет'

function createGreeter(name) {
  const prefix = '>>> '

  return function greet() {
    console.log(prefix + greeting + ', ' + name)
  }
}

const fn = createGreeter('Мир')
fn() // ">>> Привет, Мир"
```

**Шаг 1:** Создаётся глобальное окружение. В нём живут `greeting` со значением `'Привет'` и функция `createGreeter`. Это самое верхнее окружение — его `[[OuterEnv]]` равен `null`.

**Шаг 2:** Вызываем `createGreeter('Мир')`. Движок создаёт новое лексическое окружение для этого вызова. В нём: `name = 'Мир'` и `prefix = '>>> '`. Его `[[OuterEnv]]` указывает на глобальное окружение.

**Шаг 3:** Внутри `createGreeter` создаётся функция `greet`. В момент создания она запоминает текущее окружение — записывает ссылку на окружение `createGreeter` в своё внутреннее свойство `[[Environment]]`.

**Шаг 4:** `createGreeter` завершается и возвращает `greet`. Обычно после завершения функции её окружение уничтожается. Но тут особый случай — на окружение `createGreeter` есть ссылка из функции `greet`, которую мы сохранили в переменную `fn`. Поэтому окружение **остаётся в памяти**.

**Шаг 5:** Вызываем `fn()`. Создаётся окружение для `greet` — оно пустое (нет локальных переменных). Его `[[OuterEnv]]` указывает на окружение `createGreeter` (запомнили на шаге 3).

**Шаг 6:** `greet` ищет `prefix`. В своём окружении не нашла → идёт в `[[OuterEnv]]` (окружение `createGreeter`) → нашла `'>>> '`.

**Шаг 7:** Ищет `greeting`. В своём — нет. В окружении `createGreeter` — нет. Идёт ещё выше, в глобальное окружение → нашла `'Привет'`.

Вот эта цепочка `greet → createGreeter → global` и есть **scope chain** — цепочка областей видимости, по которой JavaScript ищет переменные.

## let vs var: почему это важно

### var — одна переменная на весь вызов

`var` создаёт переменную в лексическом окружении **функции**, а не блока. Это значит, что в цикле `for (var i = ...)` существует **одна** переменная `i` на весь вызов функции:

```js
function testVar() {
  for (var i = 0; i < 3; i++) {
    // Один и тот же i для всех итераций
  }
  console.log(i) // 3 — i доступна за пределами цикла!
}
```

Все замыкания, созданные внутри этого цикла, будут ссылаться на **одну и ту же** переменную `i`. К моменту их вызова `i` уже будет равна финальному значению.

### let — новая переменная на каждую итерацию

`let` работает иначе. На каждой итерации цикла JavaScript создаёт **новое** лексическое окружение блока с **новой** привязкой переменной:

```js
function testLet() {
  for (let i = 0; i < 3; i++) {
    // Каждая итерация получает свою копию i
  }
  // console.log(i) — ReferenceError! i не существует здесь
}
```

Каждое замыкание, созданное в такой итерации, захватывает свою собственную копию `i`. Именно поэтому `let` решает классическую проблему с `setTimeout` в цикле (подробнее — в следующем разделе).

## Как V8 оптимизирует замыкания

<DeepDive>

Можно подумать, что замыкание хранит **все** переменные внешней функции. На самом деле движок V8 умнее. При парсинге кода V8 анализирует, какие именно переменные внутренняя функция реально использует. Только эти переменные выделяются в специальный объект `Context`, который хранится в куче (heap). Остальные переменные остаются на стеке и освобождаются при выходе из функции. Это значит, что замыкание не «утяжеляет» память ненужными данными — V8 сохраняет только то, что действительно нужно.

</DeepDive>

## Когда окружение уничтожается

Лексическое окружение живёт в памяти, пока на него есть хотя бы одна ссылка. Как только все ссылки исчезают — сборщик мусора (Garbage Collector) его уничтожает:

```js
function create() {
  const bigData = new Array(1000000)
  return () => bigData.length
}

let fn = create() // Окружение create живёт — fn ссылается на него
fn() // 1000000
fn = null // Ссылок больше нет — GC может собрать окружение и bigData
```

Это важно для понимания утечек памяти: если замыкание живёт долго (например, обработчик событий), оно удерживает в памяти всё окружение, в котором было создано. Об утечках — подробнее в разделе про частые проблемы.

## Цепочка окружений при вложенности

Замыкания могут быть вложенными — каждая функция хранит ссылку на окружение, где была создана, образуя цепочку:

```js
function a() {
  let x = 1
  function b() {
    let y = 2
    function c() {
      let z = 3
      console.log(x + y + z) // ищет: c → b → a
    }
    return c
  }
  return b
}

a()()() // 6
```

Функция `c` ищет `z` у себя (нашла), `y` в окружении `b` (нашла), `x` в окружении `a` (нашла). Цепочка `c → b → a → global` — это и есть scope chain. Каждое звено — лексическое окружение, хранящее свои переменные и ссылку на внешнее.

## Итого

| Концепция | Описание |
|-----------|----------|
| Lexical Environment | Объект с двумя частями: переменные (Environment Record) + ссылка на внешнее окружение ([[OuterEnv]]) |
| Scope chain | Цепочка окружений от текущего к глобальному, по которой ищутся переменные |
| var vs let | var — одна привязка на функцию, let — новая на каждый блок/итерацию цикла |
| Время жизни | Окружение живёт в памяти, пока есть хотя бы одна ссылка на него |
